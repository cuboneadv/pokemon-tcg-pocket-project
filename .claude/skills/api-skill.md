# API Skill Guide

## 적용 조건 (자동 활성화)
- 키워드: API, fetch, 데이터, TCGdex, 연동, 카드 데이터, 요청, 응답
- 파일 경로: *.js (API 관련 함수)

---

## 1. 기본 원칙
- 사용 API: TCGdex API (https://api.tcgdex.net/v2)
- 임의로 다른 API 탐색/추가 금지
- 새로운 API 필요 시 반드시 사용자에게 먼저 확인

## 2. 엔드포인트 규칙
- 영어 데이터: https://api.tcgdex.net/v2/en
- 한국어 데이터: https://api.tcgdex.net/v2/ko
- TCG Pocket 시리즈: /series/tcgp
- 카드 이미지: {card.image}/high.webp
- 한국어 데이터 없을 시 영어로 자동 폴백(fallback) 처리

## 3. 데이터 요청 원칙
- 모든 API 요청은 try/catch로 에러 처리 필수
- 로딩 중 스피너 또는 스켈레톤 UI 표시
- 요청 실패 시 사용자에게 안내 메시지 표시
- 동일한 데이터 중복 요청 금지 (캐싱 적용)

## 4. 카드 이미지 원칙
- 이미지는 TCGdex CDN URL 사용
- 이미지 깨짐 방지를 위해 로컬 캐싱 고려
- 이미지 로딩 실패 시 대체 이미지 표시
- 한/영 이미지 전환: URL의 언어 파라미터만 변경

## 5. 성능 원칙
- API 응답 데이터는 localStorage에 캐싱 (24시간)
- 페이지 로드 시 캐시 우선 확인 후 API 요청
- 불필요한 API 호출 최소화

## 6. 작업 순서 원칙
- 반드시 API 테스트 먼저 → 정상 확인 후 코드 반영
- 새 엔드포인트 사용 전 반드시 콘솔에서 테스트
- 테스트 결과 사용자에게 보고 후 진행

## 7. 작업 완료 후 체크리스트
- [ ] 모든 API 요청에 에러 처리가 되어 있는가?
- [ ] 로딩 상태가 UI에 표시되는가?
- [ ] 한국어 폴백 처리가 되어 있는가?
- [ ] 캐싱이 적용되어 있는가?
- [ ] 불필요한 중복 요청이 없는가?
