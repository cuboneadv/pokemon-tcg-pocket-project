# Auth Skill Guide

## 적용 조건 (자동 활성화)
- 키워드: 로그인, 회원가입, 인증, 세션, 토큰, 비밀번호, 사용자
- 파일 경로: *.js (인증 관련 함수)

---

## 1. 기본 원칙
- 인증 라이브러리: TBD (사용자 확인 후 결정)
- 임의로 인증 방식 결정 금지
- 구현 전 반드시 사용자에게 방식 확인 후 진행

## 2. 인증 방식 옵션 (사용자 선택 필요)
- Option A: Supabase Auth (이메일/소셜 로그인, 무료)
- Option B: Firebase Auth (구글/소셜 로그인, 무료)
- Option C: 자체 구현 (JWT 기반)
- 결정 시점: 덱 빌더 기능 개발 시작 전

## 3. 로그인 기능 범위 (TBD)
- 이메일/비밀번호 로그인
- 소셜 로그인 (Google, Kakao 등)
- 비밀번호 찾기/재설정
- 회원 탈퇴

## 4. 개인화 기능 범위 (TBD)
- 덱 저장/불러오기
- 카드 즐겨찾기
- 컬렉션 트래커
- 프로필 설정

## 5. 보안 원칙
- 비밀번호 평문 저장 절대 금지
- 민감 정보 클라이언트 노출 금지
- 토큰은 httpOnly 쿠키에 저장
- 인증이 필요한 페이지는 반드시 접근 제어 적용

## 6. 작업 순서 원칙
- 인증 방식 확정 → 사용자 승인 → 구현 시작
- 단계별로 하나씩 구현 (로그인 → 회원가입 → 개인화 순서)
- 각 단계 완료 후 사용자 확인 후 다음 단계 진행

## 7. 작업 완료 후 체크리스트
- [ ] 비밀번호가 암호화되어 저장되는가?
- [ ] 민감 정보가 클라이언트에 노출되지 않는가?
- [ ] 인증이 필요한 페이지에 접근 제어가 적용되었는가?
- [ ] 로그인/로그아웃이 정상 동작하는가?
- [ ] 세션 만료 처리가 되어 있는가?
