# Security Skill Guide

## 적용 조건 (자동 활성화)
- 키워드: 보안, 취약점, XSS, CSRF, 암호화, 인증, 권한
- 파일 경로: 모든 파일 (보안은 전 영역 적용)

---

## 1. 기본 원칙
- 보안은 모든 작업에 기본으로 적용
- 보안 취약점 발견 시 즉시 사용자에게 보고
- 임의로 보안 설정 변경 금지

## 2. 프론트엔드 보안
- XSS 방지: 사용자 입력값 반드시 이스케이프 처리
- 민감 정보(API 키, DB 접속 정보) 클라이언트 코드 노출 절대 금지
- 환경 변수(.env)로 민감 정보 관리
- .env 파일 .gitignore에 반드시 추가

## 3. API 보안
- API 요청 시 인증 토큰 반드시 포함
- CORS 설정 확인 (허용된 도메인만 접근)
- API 키 하드코딩 절대 금지
- 외부 API 응답 데이터 검증 후 사용

## 4. 인증/인가 보안
- 비밀번호 평문 저장 절대 금지 (Supabase Auth 사용)
- 토큰 만료 처리 반드시 적용
- 인증이 필요한 기능은 서버 측에서도 반드시 검증
- 관리자 기능은 별도 권한 관리

## 5. 데이터베이스 보안
- Supabase RLS(Row Level Security) 반드시 적용
- 유저는 본인 데이터만 접근 가능하도록 설정
- SQL Injection 방지: Supabase SDK 파라미터 바인딩 사용
- 불필요한 데이터 노출 최소화

## 6. 배포 보안
- HTTPS 반드시 사용 (GitHub Pages 자동 지원)
- 민감 정보 GitHub에 업로드 금지
- 의존성 패키지 주기적으로 업데이트
- 사용하지 않는 코드/파일 정리

## 7. 보안 점검 타이밍
- 새 기능 추가 시
- 외부 라이브러리 추가 시
- 배포 전
- 로그인/DB 관련 작업 시

## 8. 작업 완료 후 체크리스트
- [ ] 민감 정보가 코드에 하드코딩되어 있지 않은가?
- [ ] .env 파일이 .gitignore에 추가되어 있는가?
- [ ] 사용자 입력값이 이스케이프 처리되어 있는가?
- [ ] 인증이 필요한 기능에 접근 제어가 적용되었는가?
- [ ] RLS가 올바르게 설정되어 있는가?
- [ ] HTTPS가 적용되어 있는가?
