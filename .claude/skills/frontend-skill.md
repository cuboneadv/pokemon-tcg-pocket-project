# Frontend Skill Guide

## 적용 조건 (자동 활성화)
- 키워드: UI, 디자인, 화면, 레이아웃, 카드 목록, 검색, 필터, 반응형, 스타일, 컴포넌트
- 파일 경로: *.html, *.css, *.js (UI 관련)

---

## 1. 디자인 원칙
- 기준 색상: 사용자가 참고 사이트/이미지 제공 시 확정 (TBD)
- 디자인 참고 이미지: 카드 중심의 생동감 있는 스타일
- 폰트: TBD (사용자 지정 시 업데이트)
- 모든 UI 텍스트는 한국어로 작성
- 작업 전 반드시 사용자에게 참고 디자인 확인 후 진행

## 2. 레이아웃 원칙
- 데스크탑 기준으로 작업 (기준 해상도: 1920x1080)
- 최소 지원 해상도: 1280px
- 카드 그리드: 데스크탑 4~5열 기준
- 모바일 대응은 추후 별도 작업으로 진행

## 3. 카드 UI 원칙
- 카드 이미지는 반드시 lazy loading 적용
- 이미지 로딩 실패 시 대체 이미지 표시
- 카드 hover 시 부상 애니메이션 유지
- 한/영 전환 버튼은 카드 상세 모달에 배치

## 4. 검색/필터 원칙
- 검색은 실시간 필터링 (입력 즉시 반응)
- 필터 상태는 URL 파라미터로 저장 (공유 가능)
- 검색 결과 없을 시 안내 메시지 표시

## 5. 성능 원칙
- 이미지 최적화: webp 형식 우선 사용
- 카드 목록 100개 이상 시 무한 스크롤 또는 페이지네이션 적용
- 불필요한 외부 라이브러리 추가 금지

## 6. 접근성 원칙
- 모든 이미지에 alt 텍스트 추가
- 버튼/링크에 aria-label 추가
- 키보드 네비게이션 지원

## 7. 작업 완료 후 체크리스트
- [ ] 데스크탑(1280px 이상)에서 레이아웃이 정상인가?
- [ ] 이미지 로딩 실패 처리가 되어 있는가?
- [ ] 한국어 텍스트가 올바르게 표시되는가?
- [ ] 참고 디자인 스타일이 반영되었는가?
- [ ] 불필요한 외부 라이브러리가 추가되지 않았는가?
