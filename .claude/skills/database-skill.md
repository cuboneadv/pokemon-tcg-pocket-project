# Database Skill Guide

## 적용 조건 (자동 활성화)
- 키워드: 데이터베이스, DB, 저장, 쿼리, Supabase, 테이블, 스키마
- 파일 경로: *.js (DB 관련 함수)

---

## 1. 기본 원칙
- 데이터베이스: Supabase (PostgreSQL 기반)
- 임의로 다른 DB 방식 추가 금지
- 스키마 변경 시 반드시 사용자 확인 후 진행

## 2. Supabase 활용 범위
- Database: 유저/덱/즐겨찾기/커뮤니티 데이터 저장
- Auth: 로그인/회원가입 (auth-skill.md와 연동)
- Storage: 카드 이미지 영구 보관
- Realtime: 커뮤니티 실시간 기능 (추후)

## 3. 데이터 구조 계획 (TBD)
- 유저 테이블: 회원 정보
- 덱 테이블: 덱 구성 정보
- 즐겨찾기 테이블: 카드 즐겨찾기
- 컬렉션 테이블: 보유 카드 목록
- 커뮤니티 테이블: 게시글/댓글
- 구체적 스키마: 각 기능 개발 시작 전 사용자와 협의 후 확정

## 4. 카드 데이터 관리 원칙
- 카드 원본 데이터: TCGdex API에서 fetch (DB 저장 불필요)
- 카드 이미지 기본: TCGdex CDN URL 사용
- 카드 이미지 영구 보관: Supabase Storage에 저장
- 주기적으로 TCGdex URL 유효성 검사 후 깨진 이미지 Supabase Storage로 대체

## 5. 쿼리 원칙
- 모든 쿼리는 try/catch로 에러 처리 필수
- 민감 데이터 쿼리 시 반드시 인증 확인
- 불필요한 전체 데이터 조회(SELECT *) 금지
- 페이지네이션 적용으로 대량 데이터 처리

## 6. 보안 원칙
- Supabase 접속 정보 클라이언트 노출 절대 금지
- 환경 변수(.env)로 관리
- Row Level Security(RLS) 반드시 적용
- SQL Injection 방지를 위해 Supabase SDK 파라미터 바인딩 사용

## 7. 작업 순서 원칙
- 스키마 설계 → 사용자 승인 → 구현 시작
- 각 테이블 완료 후 사용자 확인 후 다음 테이블 진행
- Supabase 대시보드에서 테이블 확인 후 코드 반영

## 8. 작업 완료 후 체크리스트

- [ ] RLS가 적용되어 있는가?
- [ ] 불필요한 전체 데이터 조회가 없는가?
