# Post-Task Hook Guide (작업 완료 후 자동 체크)

## 역할
작업이 완료되면 아래 절차를 자동으로 수행

---

## 1. 수정 파일 자동 기록
작업 중 수정/생성된 모든 파일을 .claude/logs/changes.md에 기록

### 기록 형식
```
## [날짜] [작업 유형]
- 작업 내용:
- 수정된 파일:
  - [파일 경로]: [변경 내용 요약]
- 결과: 성공/실패
- 특이사항:
```

---

## 2. 자동 품질 검사

### 2-1. 프론트엔드 작업 시
- [ ] 데스크탑(1280px 이상)에서 레이아웃 정상 여부
- [ ] 이미지 로딩 실패 처리 여부
- [ ] 한국어 텍스트 정상 표시 여부
- [ ] 불필요한 외부 라이브러리 추가 여부

### 2-2. API 작업 시
- [ ] 모든 API 요청에 에러 처리 여부
- [ ] 로딩 상태 UI 표시 여부
- [ ] 한국어 폴백 처리 여부
- [ ] 캐싱 적용 여부

### 2-3. Git 작업 시
- [ ] 커밋 메시지 규칙 준수 여부
- [ ] main 브랜치 직접 커밋 여부
- [ ] 사용자 확인 후 푸시 여부

### 2-4. Auth 작업 시
- [ ] 비밀번호 암호화 저장 여부
- [ ] 민감 정보 클라이언트 노출 여부
- [ ] 인증 필요 페이지 접근 제어 여부

### 2-5. DB 작업 시
- [ ] Supabase 접속 정보 환경 변수 관리 여부
- [ ] 모든 쿼리 에러 처리 여부
- [ ] RLS 적용 여부

### 2-6. 보안 작업 시
- [ ] 민감 정보 하드코딩 여부
- [ ] .env 파일 .gitignore 추가 여부
- [ ] 사용자 입력값 이스케이프 처리 여부

---

## 3. 오류 처리 기준

### 오류 적을 때 (1~2개)
- 즉시 자동 수정 후 사용자에게 보고

### 오류 많을 때 (3개 이상)
- 작업 중단
- 오류 목록 사용자에게 보고
- 해결 방향 제시 후 사용자 승인 후 진행

---

## 4. 체크리스트 업데이트
- .claude/plans/checklist.md 에서 완료된 항목 체크
- 남은 작업 목록 사용자에게 보고

---

## 5. 완료 후 알림 형식
작업 완료 후 아래 형식으로 사용자에게 보고:
```
✅ 작업 완료 보고
- 완료된 작업:
- 수정된 파일:
- 품질 검사 결과: 통과/수정 필요
- 수정된 오류: (있을 경우)
- 다음 작업 제안:
- GitHub 커밋 여부: 확인 후 진행하겠습니다
```
