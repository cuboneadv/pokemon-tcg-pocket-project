# Pre-Task Hook Guide (작업 시작 전 자동 체크)

## 역할
사용자 지시가 입력되면 작업 시작 전 아래 절차를 자동으로 수행

---

## 1. 지시 내용 분석
입력된 지시에서 아래 키워드를 감지하여 관련 스킬 파일 자동 로드

| 감지 키워드 | 로드할 스킬 |
|------------|------------|
| UI, 디자인, 화면, 레이아웃, 카드 목록, 검색, 필터 | frontend-skill.md |
| API, fetch, 데이터, TCGdex, 연동, 카드 데이터 | api-skill.md |
| 커밋, 브랜치, 푸시, 배포, GitHub | git-skill.md |
| 로그인, 회원가입, 인증, 세션, 토큰 | auth-skill.md |
| 데이터베이스, DB, 저장, 쿼리, Supabase | database-skill.md |
| 보안, 취약점, XSS, CSRF, 암호화 | security-skill.md |

---

## 2. 작업 시작 전 필수 확인 사항

### 2-1. 큰 작업 여부 판단
아래 중 하나라도 해당되면 → 반드시 계획 수립 후 사용자 승인 대기

- 새로운 기능 추가
- 기존 구조 변경
- 새로운 라이브러리/API 추가
- DB 스키마 변경
- 배포 관련 작업

### 2-2. 계획 수립 절차 (큰 작업 시)
1. 현재 상황 분석
2. 작업 계획 작성 후 사용자에게 보고
3. 사용자 승인 대기 (승인 전 작업 시작 금지)
4. 승인 후 아래 3개 문서 생성
   - .claude/plans/plan.md (계획서)
   - .claude/plans/context.md (맥락 노트)
   - .claude/plans/checklist.md (체크리스트)
5. 문서 생성 완료 후 작업 시작

### 2-3. 소규모 작업 여부 판단
아래 중 하나라도 해당되면 → 바로 작업 시작 가능

- 버그 수정
- 텍스트/스타일 단순 수정
- 이미 승인된 계획의 세부 작업

---

## 3. 토큰 절약 원칙 확인
- 지정된 API/라이브러리 외 임의 탐색 금지
- 불확실한 부분은 작업 전 사용자에게 질문
- 한 번에 하나의 작업만 수행
- 모든 명령은 영어로 수행

---

## 4. 시작 전 알림 형식
작업 시작 전 아래 형식으로 사용자에게 보고:
```
📋 작업 시작 전 확인
- 감지된 작업 유형: [프론트엔드/API/Git 등]
- 로드된 스킬: [skill 파일명]
- 작업 규모: [대규모/소규모]
- 계획 수립 필요: [예/아니오]
- 시작하겠습니다 / 계획을 먼저 작성하겠습니다
```
